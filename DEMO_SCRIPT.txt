[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the flow. The system starts with a Security Checkpoint that scrubs PII and checks for prompt injection attacks. Clean inputs go to the Invoice Parser Sub-Agent to extract invoice numbers, PO references, and amounts. Next, the Invoice Auditor Sub-Agent retrieves the original Purchase Order from our secure database using a custom Model Context Protocol, or MCP, server. If the auditor finds a perfect match, it goes to the Payment Agent to run the approve_payment tool. However, if there is a discrepancy, or if the invoice exceeds our five thousand dollar threshold, the Audit Router redirects the task to the Human Reviewer node. This node pauses the workflow using a RequestInput interrupt. Once a human manager approves or rejects it, the flow resumes, running either the Payment Agent or the Vendor Notifier Agent to send an automated rejection email.
